MarTeX

MarTeX is a versitale TeX to HTML interpreter. It comes with a few basic modules, but can easily be expanded upon. It was written by Thijs Miedema for use with marie-curie.nl.

Modules

Module Document

The document module exposes the basic commands that are designed to mimick the LaTeX main environment.

Implemented commands

  • global: \title{Title}

    Make a big fancy title.

  • global: \section{Title}

    Make a section title.

  • global: \subsection{Title}

    Make a subsection title.

  • global: \subsubsection{Title}

    Make a subsubsection title.

  • global: \textbf{Text}

    Render Text in bold.

  • global: \textit{Text}

    Render Text in italics.

  • global: \underline{Text}

    Render Text underlined.

  • global: \smallcaps{Text}

    Render Text smallcaps.

  • global: \hline

    Make a horizontal line.

  • global: \newline

    Line break, equivalent to a double backslash.

  • global: \paragraph{text}

    Render the text as a paragraph, normally not noticable, but might have implications if a stylesheet is loaded. Use for a single line of text without commands only, please use the environment paragraph otherwise.

  • global: \ref{label}

    Reference a label (from an image, for example). Works with both forward and backwards declaration (The label can be both above and below the ref command). Outputs a question mark when the label is undefined.

  • global: \define{Variable}{Value}

    Set the global Variable to Value. Often used to change module behaviour.

  • global: \href{URL}{Text}

    Make a hyperlink to URL with text Text

  • global: \colo(u)r{type}{definition}{text}

    Change the colour of a piece of text. There are a few possibilities: \colour{rgb}{255,0,100}{text}, \colour{rgba}{100,100,255,0.7}, \colour{hsl}{200,50,50}{text}, \colour{hsla}{200,50,50,0.7}{text} and \colour{hex}{#ff00ff}{text}. Please note that hsl and hsla are not supported by all browsers (especially email clients).

  • global: \colo(u)r{definition}{text}

    Same as above, but allow MarTeX to guess your colour type. Will always choose rgb over hsl.

Implemented environments

In MarTeX an environment is defined as the area between a begin and matching end command:

\begin{Environment} 
    Environment content
\end{Environment}


  • document:

    Does basically nothing right now, but might do in the future. For now it is there for the TeX code-style idea.

  • paragraph:

    Does the same as the command 'paragraph', but now with begin and end commands. Works better for multiline paragraphs.

  • code:

    Wrapper for the html 'pre' environment. Stuff in this environment is displayed with all whitespace intact (tabs, newlines, etc)



Example

 \define{marie}{cool}
    \title{My Website}
    \begin{document}
        \section{Chapter One}
        \subsection{Whoarewe}
        \begin{paragraph}
            \textbf{Marie Curie!} \\
            \textit{An association for students of Physics!}
        \end{paragraph}
        \subsubsection{Code!}
            \begin{code}
                    We love tabs in code!
            \end{code}       
        \ref{NotDefinedLabel}
    \end{document}


After rendering this results in:

Whoarewe

Marie Curie!
An association for students of Physics!

Code!

                    We love tabs in code!

Module Figure

The figure module exposes commands that are designed to mimick the LaTeX package graphicx, with a few exceptions. Maybe the most notable exception is that most of these commands only work inside a figure environment, and will result in "command not found" errors if you use them elsewhere.



Implemented commands

  • figure: \includegraphics{path/to/image}

    Use to display an image.

  • figure: \caption{caption text}

    Use to display text under your image.

  • figure: \label{label text}

    Use to give your image a label, so you can use \ref{label} to reference it.

  • figure: \width{Width}

    Set the width of your image. Width can have the following formats: 10px, 10%, 10cm.

  • figure: \height{Height}

    Set the height of your image. Height can have the following formats: 10px, 10%, 10cm.

  • figure: \alttext{alt text}

    Set the alt text, also called hovertext, of your image.



Settings

Reference labels are used to point to your image from the text. They put the following text in your caption: Figure N, where N is replaced with the number of your image. However, you might want to use a different text. For this you can access the global variable 'figureheader'. For example, you could do: \define{figureheader}{image}. Now, the caption wil read: Image N. You should put the \define above all figure environments.



Example

    \define{figureheader}{image}
    \begin{figure}
        \includegraphics{test.png}
        \caption{This is a test image}
        \label{testimage}
        \width{120px}
        \height{30%}
        \alttext{A mouse!}
    \end{figure}


Module Itemize

The itemize module exposes commands to help make fancy lists.



Implemented commands

  • itemize/enumerate: \item{Itemtext}

    Add an item to a list.

  • itemize/enumerate: \setmarker{type}

    Set the marker type. Valid types are: square, bullet, circle, none, numbers, letters, LETTERS, roman, ROMAN.



Implemented environments

The itemize module implements two environments. They are equivalent, but have a different default marker type.

  • itemize:

    A list with default marker 'bullet'

  • paragraph:

    A list with default marker 'numbers'

Module Tabular

Tabulars in LaTeX are not the most well defined things. This implementation is basic and subject to change. Planned features include multi-rows and columns.



Implemented commands

  • tabular: \hline

    Add a horizontal line at this position in the table. Maximum of two lines per row.

  • tabular: \linewidth

    Set the border width of your table cells.



Implemented Environments

The only environment implemented by this module is the tabular environment. It works just like the tabular in LaTeX:
\begin{tabular}{ l | c || r | }
    \hline
    1 & 2 & 3 \\ \hline
    4 & 5 & 6 \\ \hline
    7 & 8 & 9 \\
    \hline
\end{tabular}
This results in:
123
456
789

MarTeX usage

Using MarTeX for your website shenanigans is pretty easy. You start by requiring martex.php and construction a MarTeX instance. All MarTeX classes and functions are contained in the MarTeX namespace, so there won't be any collissions with any functions and classes you use yourself.

require_once (__DIR__.'/relative/path/to/martex.php');
    
$Tex = new MarTeX\MarTeX();


Then, you require the modules you like and register them to the MarTeX instance.

require_once (__DIR__.'/relative/path/to/document.php');

$Document = new MarTeX\Document();
$Tex->registerModule($Document);


Now you are all set up for TeX'in. Use the parse function with some text. This could be a file, or POST data. If you use POST data, make sure it is urldecoded and magic quotes are turned off in your php configuration. Otherwise your backslashes in your text could end up being escaped, and MarTeX will interpret them as a double backslash, resulting in a plain old newline.

$Tex->parse($my_tex_text);


We're almost finished. The last thing to do is to check for errors and maybe return them, and returning the output.

if ($Tex->hasError()) {
    echo $Tex->getError();
}
echo $Tex->getResult();


Please note that, even though we might have errors, the result is still returned. In most cases this works, since MarTeX will usually just discard commands that cause errors, or make an attempt to fix it. Only on serious syntax errors MarTeX will not return any output.

Module development

MarTeX modules are easy to develop. An empty module looks like this: 



<?php
// Add this module to the MarTeX namespace
namespace MarTeX;

// Include the base class and module tools
include __DIR__.'/module.php';

// Define the class and extend the MarTeXmodule
class Helloworld extends MarTeXmodule {

}

?>
That's all there is to it! You now have a module that does nothing...
Some remarks before we get to adding functionality to the module:
  • The filename of the module should be all lowercase.
  • The classname of the module should be the filename with the first letter capitalized.
  • The class must be defined in the MarTeX namespace.
That means we should save the example above as 'helloworld.php'.
Now for adding functionality. There are three types of ways your module can add functionality: commands, environments and previronments.

Commands

Commands are the most basic feature you can add to your module. In order to add them, you need to implement two methods in your class: registerCommands and handleCommand. registerCommands should return the commands you want to process as an array of strings. 
public function registerCommands() {
  return array("helloworld");
}
In the code example we have registered the command 'helloworld'. We now need to handle that command by implementing the 'handleCommand' function: 
public function handleCommand($command, $argument) {
  return "<h1>Hello, World!</h1>";
}
Now, when the user types \helloworld in their source, it will compile to <h1>Hello, World!</h1>! But, if the user types \helloworld{InvalidArgument}, it will also compile to the same thing! It will just ignore the argument, but we might want to give the user a warning about that. For that, we make a call to the the error raising functions, which we can access because we extended the MarTeXmodule class: 
public function handleCommand($command, $argument) {
  if (is_array($argument) or $argument != "")) {
    $this->raiseWarning("More arguments supplied to helloworld then required.");
  }
  return "<h1>Hello, World!</h1>";
}
You might notice that we first check if the argument is an array. That is because argument is an array if more then one argument is supplied, and a string when it is only a single argument.
Now, this input validation might get a bit tedious when you have multiple arguments and commands. Therefore, there is helper function for this: 
public function handleCommand($command, $argument) {
  $argument = $this->valisaniArgument($argument, 0, "");
  return "<h1>Hello, World!</h1>";
}
The valisani function takes three arguments. The first argument is input, the second number is the supposed number of arguments and the third is for defining the arguments types. This is best demonstrating by adding another command, let's try \multiply{x}{y} with x and y some number: 
public function registerCommands() {
  return array("helloworld", "multiply");
}

public function handleCommand($command, $argument) {
  switch($command) {
    case "helloworld":
        $argument = $this->valisaniArgument($argument, 0, "");
        return "<h1>Hello, World!</h1>";
    case "multiply":
        $argument = $this->valisaniArgument($argument, 2, array("Integer", "Integer"));
        return strval($argument[0] * $argument[1]);
    }
}
Normally, for the multiply to work, you'd have to check the numbers yourself, see if there are actually two numbers, cast them to numbers etc, but MarTeX does this for you and will generate warnings if the user inputs invalid arguments or doesn't supply enough of them.

Environments

Environments are areas in MarTeX source that are sandwiched between a begin and end tag. They allow you to do more complex operations on a piece of content. Environments are parsed after commands. You can add an environment to your module by implementing the registerEnvironments and handleEnvironment function.
